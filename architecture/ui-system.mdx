## Overview

The UI system (`ui.h`) provides a custom framework for building interactive interfaces with a scene graph, automatic layout computation, animations, and event handling. It's built on top of the renderer system and manages the complete UI lifecycle.

## Initialization

```c
Ui_t *ui_init(void);
void ui_finish(Ui_t *ui);
```

**`ui_init()`** creates and initializes:
- Root container (matching screen dimensions)
- Event system
- Animation system
- Layout engine

**`ui_finish()`** destroys all UI resources:
- All containers and drawables
- Active animations
- Event callbacks

<Note>
  The UI system should be initialized **after** `render_init()` as it depends on viewport information.
</Note>

## Container System

### Container Structure

```c
typedef struct Container_t {
    Bounds_t bounds;                    // Position and size
    Container_t *parent;                // Parent container
    Vector_t *child_drawables;          // Child drawables
    Vector_t *child_containers;         // Child containers
    Vector_t *animations;               // Container animations
    Layout_t layout;                    // Layout configuration
    bool enabled;                       // Visibility and interaction
    ContainerFlags_t flags;             // Layout flags
    double align_content_offset_y;      // Vertical alignment offset
    double align_content_offset_x;      // Horizontal alignment offset
    double viewport_y, viewport_x;      // Scroll viewport offsets
    bool draw_debug_overlay;            // Debug visualization
} Container_t;
```

Containers are the **organizational backbone** of the UI:
- Form a tree hierarchy (scene graph)
- Automatically compute child positions
- Can contain drawables and other containers
- Support scrolling via viewport offsets

### Creating Containers

```c
Container_t *ui_make_container(
    Ui_t *ui,
    Container_t *parent,
    const Layout_t *layout,
    ContainerFlags_t flags
);
```

**Example**:
```c
// Get root container
Container_t *root = ui_root_container(ui);

// Create a centered content area
Layout_t layout = {
    .flags = LAYOUT_CENTER,
    .width = 800,
    .height = 600
};
Container_t *content = ui_make_container(ui, root, &layout, CONTAINER_NONE);
```

### Container Flags

```c
typedef enum ContainerFlags_t {
    CONTAINER_NONE = 0,
    CONTAINER_VERTICAL_ALIGN_CONTENT = 1,    // Stack children vertically
    CONTAINER_HORIZONTAL_ALIGN_CONTENT = 2,   // Stack children horizontally
} ContainerFlags_t;
```

<Accordion title="Vertical Content Alignment">
  ```c
  Container_t *vstack = ui_make_container(
      ui, parent, &layout,
      CONTAINER_VERTICAL_ALIGN_CONTENT
  );
  ```
  
  Automatically stacks child elements vertically:
  - First child at top
  - Subsequent children below previous ones
  - Uses `align_content_offset_y` to track position
</Accordion>

<Accordion title="Horizontal Content Alignment">
  ```c
  Container_t *hstack = ui_make_container(
      ui, parent, &layout,
      CONTAINER_HORIZONTAL_ALIGN_CONTENT
  );
  ```
  
  Automatically stacks child elements horizontally:
  - First child at left
  - Subsequent children to the right
  - Uses `align_content_offset_x` to track position
</Accordion>

### Destroying Containers

```c
void ui_destroy_container(Ui_t *ui, Container_t *container);
```

Recursively destroys:
- All child containers
- All child drawables
- Associated animations
- Event callbacks

## Drawable Components

### Drawable Structure

```c
typedef struct Drawable_t {
    DrawableType_t type;            // Text, image, rect, etc.
    Texture_t *texture;             // Cached texture (if applicable)
    Bounds_t bounds;                // Position and size
    void *custom_data;              // Type-specific data
    Container_t *parent;            // Parent container
    bool enabled, dynamic;          // Visibility and caching
    Layout_t layout;                // Layout configuration
    uint8_t alpha_mod;              // Alpha (0-255)

    Vector_t *active_animations;    // Running animations

    Shadow_t *shadow;               // Optional shadow
    DrawRegionOptSet_t draw_regions; // Partial visibility

    Vector_t *events;               // Event callbacks

```

### Modifying Drawables

```c
void ui_recompute_drawable(Ui_t *ui, Drawable_t *drawable);
void ui_reposition_drawable(Ui_t *ui, Drawable_t *drawable);
void ui_destroy_drawable(Ui_t *ui, Drawable_t *drawable);
```

**`ui_recompute_drawable`**: Regenerates texture (for text changes, etc.)

**`ui_reposition_drawable`**: Recalculates position based on layout

**`ui_destroy_drawable`**: Frees all resources

## Layout System

### Layout Structure

```c
typedef struct Layout_t {
    LayoutFlags_t flags;                 // Layout mode flags
    double offset_x, offset_y;           // Position offsets
    double width, height;                // Dimensions
    double padding_w, padding_h;         // Internal padding
    const Drawable_t *relative_to_size;  // Size reference
    const Drawable_t *relative_to;       // Position reference
    SizeConstraint_t min_width, max_width;   // Width constraints
    SizeConstraint_t min_height, max_height; // Height constraints
} Layout_t;
```

### Layout Flags

```c
typedef enum LayoutFlags_t {
    LAYOUT_NONE = 0,
    
    // Centering
    LAYOUT_CENTER_X = 1 << 0,
    LAYOUT_CENTER_Y = 1 << 1,
    LAYOUT_CENTER = LAYOUT_CENTER_X | LAYOUT_CENTER_Y,
    
    // Proportional positioning (0.0 to 1.0 of parent)
    LAYOUT_PROPORTIONAL_X = 1 << 2,
    LAYOUT_PROPORTIONAL_Y = 1 << 3,
    LAYOUT_PROPORTIONAL_POS = LAYOUT_PROPORTIONAL_X | LAYOUT_PROPORTIONAL_Y,
    
    // Proportional sizing
    LAYOUT_PROPORTIONAL_W = 1 << 4,
    LAYOUT_PROPORTIONAL_H = 1 << 5,
    LAYOUT_PROPORTIONAL_SIZE = LAYOUT_PROPORTIONAL_W | LAYOUT_PROPORTIONAL_H,
    
    // Anchoring
    LAYOUT_ANCHOR_BOTTOM_Y = 1 << 6,
    LAYOUT_ANCHOR_RIGHT_X = 1 << 7,
    LAYOUT_ANCHOR_CENTER_Y = 1 << 17,
    LAYOUT_ANCHOR_CENTER_X = 1 << 18,
    
    // Special
    LAYOUT_SPECIAL_KEEP_ASPECT_RATIO = 1 << 8,
    
    // Relative positioning
    LAYOUT_RELATIVE_TO_Y = 1 << 11,
    LAYOUT_RELATIVE_TO_X = 1 << 12,
    LAYOUT_RELATIVE_TO_POS = LAYOUT_RELATIVE_TO_Y | LAYOUT_RELATIVE_TO_X,
    LAYOUT_RELATIVE_TO_HEIGHT = 1 << 13,
    LAYOUT_RELATIVE_TO_WIDTH = 1 << 14,
    LAYOUT_RELATIVE_TO_SIZE = LAYOUT_RELATIVE_TO_HEIGHT | LAYOUT_RELATIVE_TO_WIDTH,
    
    // Include size in relative calculations
    LAYOUT_RELATION_Y_INCLUDE_HEIGHT = 1 << 9,
    LAYOUT_RELATION_X_INCLUDE_WIDTH = 1 << 10,
    LAYOUT_RELATION_INCLUDE_SIZE = LAYOUT_RELATION_Y_INCLUDE_HEIGHT | LAYOUT_RELATION_X_INCLUDE_WIDTH,
    
    // Wrap around children
    LAYOUT_WRAP_AROUND_X = 1 << 15,
    LAYOUT_WRAP_AROUND_Y = 1 << 16,
    LAYOUT_WRAP_AROUND = LAYOUT_WRAP_AROUND_X | LAYOUT_WRAP_AROUND_Y,
    
    // Proportional position relative to specific element
    LAYOUT_PROPORTIONAL_X_POS_TO_RELATIVE = 1 << 19,
    LAYOUT_PROPORTIONAL_Y_POS_TO_RELATIVE = 1 << 20,
    LAYOUT_PROPORTIONAL_POS_TO_RELATIVE = LAYOUT_PROPORTIONAL_X_POS_TO_RELATIVE | LAYOUT_PROPORTIONAL_Y_POS_TO_RELATIVE,
} LayoutFlags_t;
```

### Size Constraints

```c
typedef enum SizeConstraintType_t {
    CONSTRAINT_NONE = 0,
    CONSTRAINT_ABSOLUTE = 1,  // Fixed pixel value
    CONSTRAINT_RELATIVE = 2,  // Relative to bounds
} SizeConstraintType_t;

typedef struct SizeConstraint_t {
    SizeConstraintType_t type;
    double value;

```

**Example**:
```c

    .width = 800,
    .max_width = { CONSTRAINT_RELATIVE, 0.9, &parent_bounds },  // Max 90% of parent

```

## Rendering

### Render Loop

```c
void ui_begin_loop(Ui_t *ui);
void ui_draw(const Ui_t *ui);
void ui_end_loop(void);
```

**Integration**:
```c
while (running) {
    render_clear();
    
    ui_begin_loop(ui);  // Prepare UI for rendering
    ui_draw(ui);        // Draw all elements
    ui_end_loop();      // Finalize rendering
    
    render_present();
}
```

**`ui_begin_loop`**:
- Updates animations
- Processes events
- Recomputes layouts if needed

**`ui_draw`**:
- Traverses scene graph
- Draws all enabled containers and drawables
- Applies transformations and effects

**`ui_end_loop`**:
- Cleans up temporary state
- Frees destroyed elements

## Best Practices

<Note>
  **Performance**: Use `dynamic = false` for static drawables. Dynamic drawables regenerate textures every frame.
</Note>

<Info>
  **Layout changes**: Call `ui_reposition_drawable()` or `ui_recompute_drawable()` when manually changing layout properties.
</Info>

**Memory management**:
- Destroy unused drawables/containers to free GPU memory
- Reuse textures when possible
- Use null textures for dynamic content

**Animation conflicts**:
- Use `AnimationApplyType_t` to control concurrent animations
- Avoid animating the same property from multiple sources
