The karaoke mode module (`karaoke.h`) manages the core karaoke experience: displaying synchronized lyrics with dynamic fill effects, visual enhancements, reading hints, and album artwork.

## Overview

Karaoke mode (`APP_MODE_KARAOKE`) is one of the primary operating modes of Etsuko. It provides:

- **Real-time lyric display** synchronized with audio playback
- **Character-by-character fill effects** based on song timings
- **Visual effects** including pulse, enlargement, and color transitions
- **Reading hints** (furigana) for pronunciation guidance
- **Album art rendering** with playback controls
- **Progress tracking** with time display and seekable progress bar

## Real-Time Lyric Display

The core feature of karaoke mode is synchronized lyric display.

### Timing Synchronization

**Process:**

1. **Get current playback time** from audio system
2. **Apply time offset** from song configuration (`Song_t.time_offset`)
3. **Find active lyric lines** where `current_time >= base_start_time`
4. **Calculate line progress** for fill effects

```c
// Conceptual synchronization
double current_time = audio_get_position() + song->time_offset;

for (size_t i = 0; i < song->lyrics_lines->size; i++) {
    Song_Line_t *line = song->lyrics_lines->data[i];
    
    if (current_time >= line->base_start_time &&
        current_time < line->base_start_time + line->base_duration) {
        // Line is currently active
        double line_progress = current_time - line->base_start_time;
        render_lyric_line_with_fill(line, line_progress);
    }
}
```

## Dynamic Fill Effects

### Character-by-Character Fill

When songs include sub-timings (`Song_t.has_sub_timings == true`), lyrics fill character-by-character:

**Implementation approach:**

1. **Iterate through timing segments** in active line
2. **Calculate cumulative time** using `Song_LineTiming_t.cumulative_duration`
3. **Determine fill progress** for current segment
4. **Render partial character** with color transition

```c
// Conceptual fill rendering
for (int i = 0; i < line->num_timings; i++) {
    Song_LineTiming_t *timing = &line->timings[i];
    double segment_start = timing->cumulative_duration;
    double segment_end = segment_start + timing->duration;
    
    if (line_progress >= segment_end) {
        // Segment fully filled
        render_text_segment(line, timing, FILLED_COLOR);
    } else if (line_progress >= segment_start) {
        // Segment partially filled
        double fill_ratio = (line_progress - segment_start) / timing->duration;
        render_text_segment_partial(line, timing, fill_ratio);
    } else {
        // Segment not yet active
        render_text_segment(line, timing, UNFILLED_COLOR);
    }
}
```

### Fill Modes

<Info>
The `assume_full_sub_timing_when_absent` flag enables linear fill effects even when explicit sub-timings aren't provided by adding a single timing covering the entire line.
</Info>

**Available fill modes:**
- **Linear fill** - Smooth left-to-right fill based on line duration
- **Full-word fill** - Entire words/segments fill at once (requires sub-timings)
- **Character fill** - Individual character fill (requires character-level timings)


### Enlargement Effect

The currently singing line appears larger than surrounding lines:


### Color Transitions

- **Unfilled** - Dimmed or outlined text
- **Filling** - Gradient from unfilled to filled color
- **Filled** - Full brightness/color

## Reading Hints Display

For songs with reading information (`Song_t.has_reading_info == true`):

```c
// Reading hints from Song_LineReading_t
typedef struct Song_LineReading_t {
    size_t start_ch_idx, end_ch_idx;
    char *reading_text;
} Song_LineReading_t;
```

**Display:**
- Reading hints render **above** their corresponding text
- Positioned based on character indices (`start_ch_idx`, `end_ch_idx`)
- Smaller font size than main lyrics
- Synchronized with lyric fill state

**Example (Japanese furigana):**
```
    よ
漢字 → 読み方
```

## Performance Considerations

<Note>
**Rendering optimization:** Only visible lyric lines are rendered. Lines far from the current playback position are skipped to maintain performance.
</Note>

**Best practices:**

- **Text caching** - Pre-render static text elements
- **Partial updates** - Only update changing elements (progress bar, current line)
- **Culling** - Skip rendering off-screen or inactive lines
- **Resource preloading** - Load all assets during `karaoke_load_loop()` before playback starts
